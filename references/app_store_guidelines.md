# App Store Guidelines Reference

## Screenshot Requirements

### iPhone Screenshot Specifications

#### 6.7" Display (Required)
**Devices:** iPhone 16 Pro Max, iPhone 15 Pro Max, iPhone 14 Pro Max
- **Portrait:** 1290 x 2796 pixels
- **Landscape:** 2796 x 1290 pixels
- **Status:** Required for iPhone apps

#### 6.5" Display (Required)
**Devices:** iPhone 14 Plus, iPhone 13 Pro Max, iPhone 12 Pro Max, iPhone 11 Pro Max, iPhone 11
- **Portrait:** 1284 x 2778 pixels
- **Landscape:** 2778 x 1284 pixels
- **Status:** Required for iPhone apps

#### 5.5" Display (Optional but Recommended)
**Devices:** iPhone 8 Plus, iPhone 7 Plus, iPhone 6s Plus
- **Portrait:** 1242 x 2208 pixels
- **Landscape:** 2208 x 1242 pixels
- **Status:** Optional but highly recommended

### iPad Screenshot Specifications

#### iPad Pro 12.9" (6th Gen)
- **Portrait:** 2048 x 2732 pixels
- **Landscape:** 2732 x 2048 pixels
- **Status:** Required if supporting iPad

#### iPad Pro 12.9" (2nd Gen)
- **Portrait:** 2048 x 2732 pixels
- **Landscape:** 2732 x 2048 pixels
- **Status:** Required if supporting iPad

### Technical Requirements

**Format:**
- High-quality JPEG or PNG
- RGB color space
- 72 dpi minimum
- Flattened (no transparency in JPEG)

**Content:**
- No transparency (except PNG)
- No alpha channel issues
- No status bar (unless it's part of design)
- No device frame (unless using Apple template)

**Quality:**
- No upscaled images
- No blurry or pixelated content
- Professional appearance

## Prohibited Content

### Metadata Restrictions

**Cannot Include:**
- ❌ Pricing information ("Free", "$0.99")
- ❌ Platform references ("iOS app", "iPhone app")
- ❌ Version numbers in title
- ❌ Emoji in app name or subtitle
- ❌ Special characters that aren't part of brand
- ❌ Competitor app names (trademark violation)
- ❌ "Best", "#1", "Top" (unless awarded by Apple)
- ❌ "Coming Soon" or future promises
- ❌ Unverifiable claims ("fastest", "best")

**Screenshot Restrictions:**
- ❌ Beta/Alpha labels
- ❌ Test data ("Test User", "123 Main St")
- ❌ Debug information
- ❌ Android or other platform UI
- ❌ Promises of future features
- ❌ Pricing or promotional offers
- ❌ Third-party trademarks without permission

**Description Restrictions:**
- ❌ HTML (use markdown only)
- ❌ Excessive punctuation (!!!, ???)
- ❌ All caps (except acronyms)
- ❌ URLs (except at end for support)
- ❌ Email addresses in main text

### Keyword Field Restrictions

**Not Allowed:**
- ❌ Competitor app names
- ❌ Irrelevant keywords
- ❌ Trademarked terms you don't own
- ❌ Adult/explicit terms (unless 17+ app)
- ❌ Category names ("games", "productivity")
- ❌ Plural forms if singular is in title (and vice versa)
- ❌ Spaces after commas (wastes characters)

**Best Practices:**
- ✅ Include both US and UK spellings if relevant
- ✅ Add common misspellings
- ✅ Use all 100 characters
- ✅ Research what users actually search

## Review Guidelines

### App Store Review Guidelines Overview

**Five Key Pillosophies:**
1. Safety - Users should feel safe
2. Performance - Apps should work well
3. Business - Clear business models
4. Design - Apple-quality design
5. Legal - Comply with laws

### Common Rejection Reasons

**Performance Issues:**
- Crashes or bugs
- Broken links
- Placeholder content
- Incomplete functionality
- Slow load times

**Business Issues:**
- Misleading pricing
- Hidden costs
- No alternative to IAP for digital goods
- Gambling without proper licensing

**Design Issues:**
- Poor user interface
- Copying other apps
- Not using native controls when appropriate
- Excessive advertisements

**Content Issues:**
- Offensive content
- Objectionable content
- User-generated content without moderation
- Copyright infringement

### Preparing for Review

**Checklist:**
- [ ] Test on actual devices
- [ ] Check all functionality works
- [ ] Verify no placeholder text/images
- [ ] Test on different iOS versions
- [ ] Ensure backend services are live
- [ ] Check all links work
- [ ] Verify IAP works (if applicable)
- [ ] Test app in different regions

**Demo Account:**
- Provide if app requires login
- Include in App Review Information
- Ensure account has premium features enabled
- Include clear setup instructions

**Contact Information:**
- Accurate phone number
- Monitored email address
- Responsive during review period

### Expedited Review

**When to Request:**
- Critical bug fix
- Time-sensitive content
- Security vulnerability fix

**How to Request:**
- App Store Connect > App Information
- "Request Expedited Review"
- Explain urgency clearly
- Use sparingly (1-2 times per year)

**Not Guaranteed:**
- Apple reviews all requests
- May still take 24-48 hours
- Don't plan launches around it

## App Preview Video Guidelines

### Technical Specifications

**Format:**
- MOV, M4V, or MP4
- Up to 500MB
- H.264 or ProRes 422

**Duration:**
- 15-30 seconds recommended
- Maximum 30 seconds

**Dimensions:**
- Same as screenshots for each device
- Native resolution required

### Content Guidelines

**Must Include:**
- ✅ Actual app footage (80%+ of video)
- ✅ First-time user experience
- ✅ Clear, in-focus footage

**Cannot Include:**
- ❌ Hands or device frames
- ❌ Overlaid device frames
- ❌ External footage (people using app)
- ❌ Animations not from app
- ❌ Graphic overlays with excessive text

**Audio:**
- Optional but recommended
- Must be original or properly licensed
- No loud or jarring sounds

### Poster Frame

- Select carefully (shows before play)
- Should entice users to watch
- Can be different from screenshots

## Localization Guidelines

### Supported Languages

**Tier 1 (Full Support):**
- English (US, UK, AU, CA)
- Chinese Simplified
- Chinese Traditional
- Japanese
- Korean
- French
- German
- Spanish (Spain, Mexico)
- Portuguese (Brazil)
- Russian

**Tier 2 (Growing Markets):**
- Arabic
- Hindi
- Indonesian
- Italian
- Dutch
- Swedish
- Thai
- Turkish
- Vietnamese

### Localization Best Practices

**App Name:**
- Can stay in English for brand recognition
- Or translate if name has meaning
- Test how it appears in search

**Keywords:**
- Never direct translate
- Research local search behavior
- Use native speaker review

**Screenshots:**
- Translate all text overlays
- Adapt imagery for culture
- Consider text expansion (some languages longer)

**Description:**
- Professional translation (not machine)
- Adapt cultural references
- Localize URLs if possible

## Kids Category Requirements

### Additional Guidelines

**Content:**
- No advertising
- No analytics without parental gate
- No in-app purchases without parental gate
- No links outside app without parental gate
- No social sharing without parental gate

**Design:**
- Simple, intuitive interface
- Age-appropriate content
- No data collection without consent
- Clear privacy policy

**Parental Gates:**
- Required for actions that exit app
- Must require adult-level action
- Not just simple math or swiping

## Subscription Guidelines

### Required Elements

**Clear Value:**
- Explain what user gets
- Before purchase screen
- In description

**Pricing Clarity:**
- Show billing frequency
- Show trial details
- Show renewal terms

**Easy Cancellation:**
- Clear how to cancel
- Link to subscription management
- Not hidden or buried

**Privacy:**
- Clear data usage
- What happens if unsubscribed
- Data retention policy

## App Store Connect Checklist

### Before Submitting

**App Information:**
- [ ] Name (30 chars)
- [ ] Subtitle (30 chars)
- [ ] Privacy Policy URL
- [ ] Category (Primary and Secondary)

**Pricing and Availability:**
- [ ] Price tier selected
- [ ] Territories selected
- [ ] Pre-order configured (if applicable)

**App Privacy:**
- [ ] Privacy nutrition label complete
- [ ] Data collection disclosed
- [ ] Privacy policy URL valid

**Version Information:**
- [ ] Screenshots for all sizes
- [ ] App Preview (optional)
- [ ] Description (4000 chars)
- [ ] Keywords (100 chars)
- [ ] Support URL
- [ ] Marketing URL (optional)
- [ ] What's New text

**Build:**
- [ ] Build uploaded and processed
- [ ] No processing errors
- [ ] Build selected for submission

**App Review Information:**
- [ ] Contact information
- [ ] Demo account (if needed)
- [ ] Notes for reviewer
- [ ] Attachment (screenshots of issues)

### Version Release Options

**Manual Release:**
- Control exactly when app goes live
- Recommended for coordinated launches
- Review approval doesn't auto-publish

**Automatic Release:**
- Goes live immediately after approval
- Good for routine updates
- Less control over timing

**Scheduled Release:**
- Set specific date/time
- Must be at least 24 hours out
- Can change until approved
